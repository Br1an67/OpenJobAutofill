# OpenJobAutofill

OpenJobAutofill 是一个 Brave/Chrome Manifest V3 扩展，用本机简历资料辅助填写招聘网站表单。它会扫描当前页面字段，生成可确认的填充草稿，并在用户确认后把匹配到的内容写入页面。

核心原则是隐私优先：简历资料保存在本机，AI 只用于理解页面字段和资料字段目录，不接收具体简历值；插件不会自动提交表单。

## 主要功能

- 本机维护简历资料，设置页用结构化表单填写，不需要用户编辑 JSON。
- 一键扫描当前招聘表单，生成填充草稿，用户确认后再写入页面。
- 支持常见输入框、文本域、单选/复选、下拉框、日期控件和部分组件库控件。
- 内置 zhiye、HotJob、Ant Design、Element UI 等常见招聘表单/组件的启发式适配。
- 右侧资料面板可查看分类资料、搜索字段和值、手动复制内容。
- 可选接入 OpenAI 兼容 API 或自定义 API，用于提升页面字段理解能力。
- 支持刷新模型候选、测试 API 连接、手动输入任意模型名。
- 资料只保存在 `chrome.storage.local`，不会同步到云端。
- 支持导入和导出 OpenJobAutofill 资料备份，方便迁移到同一浏览器的其他配置。
- 页面填写结果会用颜色标记，方便人工复核。

## 适合场景

- 校招、社招、实习申请里重复填写个人信息、教育经历、实习经历、项目经历等字段。
- 不同公司表单字段名称不完全一致，但大致属于通用简历信息。
- 用户希望 AI 帮忙理解页面结构，但不希望把简历具体内容发送给模型服务。

## 不适合场景

- 需要自动点击最终提交按钮的全自动投递流程。
- 高度定制、强校验、跨多页强状态依赖的复杂系统。
- 需要把开放题回答、求职信、岗位定制内容直接交给 AI 生成的场景。

## 安装到 Brave 或 Chrome

1. 打开 `brave://extensions/` 或 `chrome://extensions/`。
2. 开启右上角 `Developer mode`。
3. 点击 `Load unpacked`。
4. 选择本项目目录：`OpenJobAutofill`。
5. 固定扩展图标，方便在招聘页面打开弹窗。

本项目当前不需要 `npm install` 或构建步骤，直接加载项目根目录即可。

## 首次配置

1. 点击扩展图标，进入 `设置`。
2. 在 `简历资料` 区域按模块填写资料，例如 `基本信息`、`教育经历`、`实习经历`、`项目经历/实践活动`。
3. 点击 `保存资料`，资料会写入本机 `chrome.storage.local`。
4. 如需测试资料，可以导入项目根目录的 `sample-profile.json`。
5. 如需 AI 页面分析，在 `API 设置` 中填写 API Key、Base URL、Endpoint Path 和模型名。
6. 点击 `测试连接` 确认 API 可用，必要时点击 `刷新模型列表` 获取候选模型。

API 是可选项。不配置 API 时，插件仍会使用本地规则尝试识别和填写字段。

## 使用流程

1. 打开招聘网站的简历填写或申请表单页面。
2. 点击 OpenJobAutofill 扩展图标。
3. 点击 `开始填写`。
4. 等待页面右侧显示扫描和匹配进度，不要重复点击。
5. 进入填充草稿后，检查每一项匹配结果和勾选状态。
6. 点击确认写入，插件会把选中的项目填入当前页面。
7. 检查页面上的颜色标记和实际字段内容。
8. 对没有识别准的字段，打开资料面板搜索或进入对应分类手动复制。
9. 最后由用户自己确认页面内容并手动提交。

## 资料模块建议

设置页会提供结构化表单，用户不需要直接编辑 JSON 或资料文件。保存时会自动生成 OpenJobAutofill 的本机资料备份内容，侧边栏和填充逻辑都会读取这份资料。

推荐按通用网申在线简历模块组织资料：`基本信息`、`求职意向`、`教育经历`、`实习经历`、`工作经历`、`项目经历/实践活动`、`干部任职经历（在校职务）`、`奖惩情况`、`外语能力`、`计算机技能（IT技能）`、`证书`、`家庭情况`、`培训经历`、`论文和著作`、`专利`、`自我描述`、`有关声明`、`其他信息`。

如果某些网站出现特殊字段，可以在对应模块里使用 `添加自定义字段` 补充字段名和值。导出的资料备份主要用于迁移或恢复，不建议手动编辑。

## AI 与隐私边界

- 简历资料值只保存在本机 `chrome.storage.local`。
- AI 请求只包含当前页面字段结构和本地资料字段目录，不包含具体简历值。
- API Key 只保存在本机扩展存储里。
- 插件不会自动提交申请表单。
- 写入页面前会生成草稿，由用户选择哪些项目可以写入。
- 页面写入后仍需要用户人工检查，尤其是证件、联系方式、日期、选择题和声明类字段。

## 权限说明

- `activeTab`：在用户点击扩展后访问当前标签页，用于扫描和填写当前页面。
- `scripting`：向当前页面注入或执行扩展脚本。
- `storage`：保存本机简历资料、API 配置和侧边栏状态。
- `<all_urls>`：招聘表单可能分布在不同域名，扩展需要在用户打开的招聘页面运行内容脚本。

## 填写结果说明

- 绿色标记表示插件已经成功写入。
- 黄色标记表示需要人工确认或匹配不够确定。
- 红色标记表示写入失败或控件不支持。
- 如果页面被刷新、切换步骤或动态加载，建议重新点击 `开始填写`。

## 常见问题

### 点击开始填写没有反应

先刷新目标页面，再重新打开扩展弹窗。如果仍无反应，回到 `brave://extensions/` 或 `chrome://extensions/` 重载扩展。

### API 测试成功但填写效果一般

API 只帮助理解页面字段，不直接接收简历值，也不会替代本地规则。建议先检查本机资料里是否有对应字段名和值，例如同一个手机号可以同时写 `电话`、`手机号码`。

### 下拉框或日期控件填不进去

不同网站控件实现差异很大。可以先用资料面板搜索对应内容，手动选择页面控件；如果同一类控件经常失败，再在代码里补充通用适配。

### 多页表单怎么处理

当前建议每进入一个新页面或新步骤后，重新点击 `开始填写`。插件不会自动跨页继续填写，也不会自动提交。

### 如何清空本机资料

进入设置页，点击 `清空本地数据`。这会删除本机保存的简历资料和配置。

## 开发说明

项目结构：

```text
OpenJobAutofill/
  manifest.json
  sample-profile.json
  src/
    background.js   # 本地存储、API 调用、模型列表和连接测试
    content.js      # 页面扫描、字段匹配、草稿生成、写入和右侧资料面板
    popup.html      # 扩展弹窗入口
    popup.js
    popup.css
    options.html    # 设置页
    options.js
    options.css
```

常用检查命令：

```bash
node --check src/content.js
node --check src/options.js
node --check src/background.js
node --check src/popup.js
git diff --check
```

## 当前限制

- 页面识别依赖 DOM 标签、可见文本、控件属性和可选 AI 分析，无法保证所有网站都一次填准。
- 动态弹窗、复杂级联选择器、强校验日期控件仍可能需要人工处理。
- 声明类、合规类、开放题字段建议人工确认，不建议盲填。
- 本项目当前没有自动更新机制，开发者模式安装后需要手动重新加载扩展。
